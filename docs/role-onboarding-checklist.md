# Role Onboarding Checklist

**Purpose:** Ensure new team members understand their role, responsibilities, and key interactions from day one. This checklist addresses onboarding gaps and establishes clear accountability from the start.

**Relates to:** Issue #4 - Process improvements to prevent loss of accountability

---

## General Onboarding (All Roles)

**Pre-Day 1:**
- [ ] Access to GitHub repository granted
- [ ] Added to relevant team channels (Slack/Teams)
- [ ] Calendar invites sent for recurring meetings
- [ ] Laptop/equipment setup completed
- [ ] Email and organizational accounts active

**Week 1:**
- [ ] Review [Project Management Overview](octoacme-project-management-overview.md)
- [ ] Read [Roles and Personas](octoacme-roles-and-personas.md) document
- [ ] Identify your primary role and responsibilities
- [ ] Meet with manager for role expectations discussion
- [ ] Introduce yourself to the team
- [ ] Review current project board and active work
- [ ] Understand escalation paths and communication cadence

**Week 2:**
- [ ] Attend key recurring meetings (standups, planning, etc.)
- [ ] Shadow a team member in your role (if applicable)
- [ ] Review relevant phase documentation based on current project phase
- [ ] Understand Definition of Done for your work
- [ ] Set up development environment (for technical roles)

**First Month:**
- [ ] Complete first meaningful contribution
- [ ] Provide onboarding feedback to improve this checklist
- [ ] Schedule 1:1 check-in with manager

## Role-Specific Onboarding

### Product Owner

**Additional Reading:**
- [ ] Review product roadmap and vision documents
- [ ] Review current product backlog
- [ ] Read [Project Initiation Guide](octoacme-project-initiation.md)
- [ ] Read [Project Planning Guide](octoacme-project-planning.md)

**Key Activities:**
- [ ] Meet with outgoing Product Owner (if transition)
- [ ] Meet with Product Manager to align on strategy
- [ ] Meet with key Stakeholders to understand priorities
- [ ] Review historical sprint velocity and team capacity
- [ ] Attend backlog refinement and sprint planning sessions
- [ ] Review acceptance criteria standards with QA

**Key Handoffs to Understand:**
- [ ] How Business Analyst provides requirements analysis
- [ ] How to collaborate with Product Manager on roadmap
- [ ] How to work with Scrum Master on backlog health
- [ ] When and how to approve/reject completed work

---

### Scrum Master

**Additional Reading:**
- [ ] Review team's working agreements
- [ ] Review retrospective history and action items
- [ ] Read [Execution & Tracking Guide](octoacme-execution-and-tracking.md)
- [ ] Read [Retrospective Guide](octoacme-retrospective-and-continuous-improvement.md)

**Key Activities:**
- [ ] Meet with outgoing Scrum Master (if transition)
- [ ] Review team metrics (velocity, burndown charts)
- [ ] Observe all Scrum ceremonies before facilitating
- [ ] Identify current team impediments
- [ ] Meet with Product Owner to understand backlog priorities
- [ ] Review escalation procedures with Project Manager

**Key Handoffs to Understand:**
- [ ] How to coordinate with Project Manager on cross-team issues
- [ ] When to escalate vs. resolve impediments directly
- [ ] How to protect team capacity with Product Owner
- [ ] Release coordination touchpoints with Release Manager

---

### Stakeholder

**Additional Reading:**
- [ ] Review Project One-pager and current project charter
- [ ] Read [Risk Management & Communication](octoacme-risks-and-communication.md)
- [ ] Review monthly stakeholder update templates

**Key Activities:**
- [ ] Meet with Project Manager for project overview
- [ ] Understand your decision authority and approval gates
- [ ] Review upcoming milestones and demo schedule
- [ ] Clarify communication preferences and cadence
- [ ] Identify other stakeholders and their areas of interest

**Key Handoffs to Understand:**
- [ ] When and how to provide input to Product Owner
- [ ] How to escalate business-critical concerns
- [ ] What information you'll receive and when
- [ ] Your role in go/no-go decisions

---

### Business Analyst

**Additional Reading:**
- [ ] Review existing requirements documentation
- [ ] Review backlog items and user story format
- [ ] Read [Project Planning Guide](octoacme-project-planning.md)
- [ ] Review acceptance criteria examples

**Key Activities:**
- [ ] Meet with Product Owner to understand product vision
- [ ] Meet with Stakeholders to understand business context
- [ ] Review requirements gathering tools and templates
- [ ] Shadow requirements workshops or stakeholder interviews
- [ ] Review process flow documentation standards
- [ ] Understand the Definition of Ready for backlog items

**Key Handoffs to Understand:**
- [ ] How to receive inputs from Product Owner and Stakeholders
- [ ] How to deliver requirements to Developers
- [ ] How to collaborate with QA on acceptance criteria
- [ ] When to engage with Project Manager on scope risks

---

### Release Manager

**Additional Reading:**
- [ ] Review [Release & Deployment Guide](octoacme-release-and-deployment.md)
- [ ] Review release runbooks and rollback procedures
- [ ] Review release history and post-mortems
- [ ] Understand deployment pipeline and infrastructure

**Key Activities:**
- [ ] Meet with outgoing Release Manager (if transition)
- [ ] Review release calendar and upcoming deployments
- [ ] Understand staging and production environments
- [ ] Review monitoring and alerting systems
- [ ] Meet with QA to understand testing and sign-off process
- [ ] Review change management and approval workflows
- [ ] Shadow a production deployment

**Key Handoffs to Understand:**
- [ ] How QA provides release sign-off
- [ ] How to coordinate with Developers on code freeze
- [ ] When to involve Project Manager on release risks
- [ ] How to communicate with Stakeholders about releases
- [ ] Escalation procedures for deployment issues

### Developer

**Additional Reading:**
- [ ] Review codebase README and architecture docs
- [ ] Review coding standards and PR guidelines

- [ ] Review CI/CD pipeline and testing requirements

**Key Activities:**
- [ ] Set up local development environment
- [ ] Review Definition of Done and PR standards
- [ ] Complete a small starter task or bug fix
- [ ] Shadow code review process
- [ ] Understand branching strategy and git workflow

**Key Handoffs to Understand:**
- [ ] How to receive requirements from Business Analyst
- [ ] How to collaborate with QA on test coverage
- [ ] When to engage with Release Manager on deployments
- [ ] How to participate in estimation and planning

**Additional Reading:**
- [ ] Review test plans and test case repositories
- [ ] Review Definition of Done and QA standards

- [ ] Read [Release & Deployment Guide](octoacme-release-and-deployment.md)

**Key Activities:**
- [ ] Set up test environments and access
- [ ] Review test automation framework (if applicable)
- [ ] Understand regression testing procedures
- [ ] Shadow a sprint review or demo
- [ ] Review defect tracking and triage process
- [ ] Understand release sign-off criteria

**Key Handoffs to Understand:**
- [ ] How to receive acceptance criteria from Business Analyst
- [ ] How to coordinate testing with Developers
- [ ] When to provide sign-off to Release Manager
- [ ] How to escalate quality risks to Project Manager

## Onboarding Completion

**Manager Review:**
- [ ] Manager confirms role responsibilities are understood
- [ ] First deliverable reviewed and feedback provided
- [ ] Questions and concerns addressed
- [ ] Onboarding marked complete in HR system

**New Team Member Feedback:**
- [ ] What went well in onboarding?
- [ ] What could be improved?
- [ ] Any missing information or resources?
- [ ] Suggestions for this checklist?
